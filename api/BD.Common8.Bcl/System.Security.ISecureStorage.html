为键/值对提供简单的安全存储(值类型泛型与非泛型重载方法不兼容)

仅客户端：如果需要存储明文数据或需要同步方法，则可使用首选项(Preferences2)

Remarks

每个平台使用平台提供的本机 API 来安全存储数据：

  • iOS：数据存储在 KeyChain 中。有关 SecAccessible 的其他信息
  • Android：加密密钥存储在密钥库中，加密数据存储在命名的共享首选项容器中
  • UWP：数据使用 DataProtectionProvider 加密并存储在命名的 ApplicationDataContainer
  • Windows 10 Desktop：数据使用 DataProtectionProvider 加密并存储在仓储中
  • Other Desktop：数据使用 AES 加密并存储在仓储中
  • AspNetCore：数据存储在仓储中

注意：在运行 Android 23（6.0/M）以下的 Android 设备上，KeyStore 中没有 AES 可用。作为最佳实践，此 API 将生成存储在 KeyStore 中的 RSA/ECB/PKCS7Padding 密钥对（KeyStore 中这些较低的API级别支持的唯一类型），用于包装运行时生成的 AES 密钥。这个包装好的密钥存储在首选项中
